# Contributing

Thanks for helping out! Here's everything you need to know.

## Prerequisites

- [.NET 8 SDK](https://dotnet.microsoft.com/download)
- Terraria + tModLoader installed (the build references tML assemblies automatically)
- No special IDE required; VS Code or Rider work well

## Build

From the repo root:

```
dotnet build MerchantsPlus.csproj /p:BuildMod=false
```

Always use `/p:BuildMod=false` — full packaging can fail when tModLoader has files locked. A successful build prints `0 Error(s)` and `0 Warning(s)`.

## Project Structure (Quick Map)

```
Shops/                  Runtime shop logic (Shop.cs, ShopMerchant.cs, …)
Shops/Catalogs/         Item lists per merchant — edit these for content changes
  ShopExpandedCatalog.cs   ← AUTO-GENERATED, do not edit by hand
```

## The Golden Rule

**Never hand-edit `Shops/Catalogs/ShopExpandedCatalog.cs`.** It is generated by `MerchantsPlus.Generator` (a C# project). Any manual edits will be overwritten on the next generation run.

## Adding or Changing Shop Content

1. Edit `../MerchantsPlus.Generator/ItemClassifier.cs` (shop groupings) or `CatalogGenerator.cs` (pipeline/rules).
2. Regenerate (run from `MerchantsPlus/` mod root):
   ```
   dotnet run --project ../MerchantsPlus.Generator
   ```
3. Build and test.

See `../MerchantsPlus.Generator/README.md` for full generator docs.

## Rules

- Use `ItemID.*` names — never raw numeric IDs.
- Max **40 items** per shop page.
- Keep item lists inside `Shops/Catalogs/`, not scattered across runtime files.
- No raw coin items in expanded coin groups.
- No trapped chests in generated shops.

## Submitting Changes

Open a pull request against `main`. Include a brief description of what changed and why. For catalog changes, include the regenerated `ShopExpandedCatalog.cs`.
