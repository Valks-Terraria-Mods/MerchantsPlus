# MerchantsPlus

MerchantsPlus expands Terraria merchant inventories with progression-gated shops.

The mod is designed to make item access broad but still gradual: shops unlock as you progress through bosses, events, and other unlock conditions instead of dumping everything at game start.

## What Is Included

- Expanded shops for vanilla town NPC merchants.
- Expanded-only shop support for additional NPCs, including:
    - `Golfer`, `Princess`
    - Town pets: `Cat`, `Dog`, `Bunny`
    - Town slimes: `Nerdy`, `Cool`, `Elder`, `Clumsy`, `Diva`, `Surly`, `Mystic`, `Squire`
- Generated expanded catalog coverage driven by `TerrariaItemListComplete1.4.5.txt` + classification data.
- Progression-aware shop visibility so locked-empty pages do not appear.
- Minimum sell-price safety for no-value items (auto 1 copper when needed and no custom price exists).

## Shop Flow In-Game

- Interacting with a supported merchant opens the modded shop panel.
- The panel lists only currently visible shops for that merchant.
- Selecting a shop opens that shop inventory.
- Shop hints show progression guidance for next unlocks.

## World Merchant Browser

Use the `World Shop Catalog Item` to browse shops independent of NPC proximity.

- Item file: `Items/WorldShopCatalogItem.cs`
- Recipe: `5x Wood`
- Use behavior: left-click use opens `World Merchant Shops` UI.
- The world browser shows only merchants currently present in the world.

## Progression And Catalog Design

- Expanded shop items are centralized in catalog classes.
- `Shops/Catalogs/ShopExpandedCatalog.cs` is generated by PowerShell.
- Base/non-expanded logic stays in `Shops/Shop*.cs`.
- Page size is capped at `40` items.

## Config Highlights

Config is server-side (`Config.cs`).

- `DevMode`: enables developer UI controls.
- `ShowAllItems`: hidden toggle used by UI when `DevMode` is enabled.
- `UnlockAllItems`: effective flag (`ForceUnlockAllItems` or `DevMode && ShowAllItems`) that bypasses progression gating.
- Additional gameplay toggles include drops, scaling, projectiles, and happiness multiplier behavior.

## Build

Compile-only build (recommended during active development):

```powershell
dotnet build /p:BuildMod=false
```

If `tModLoader.targets` is missing, launch tModLoader once and build any mod, then retry.

## Related Scripts

- `generate_expanded_catalog.ps1`
    - Generates `Shops/Catalogs/ShopExpandedCatalog.cs`.
- `classify_missing.ps1`
    - Classifies missing items by stage into `.tmp/missing_classified.csv`.
- `.tmp/generate_expanded_catalog.ps1`
    - Wrapper that forwards to root generator.
